# Motivations

I couldn't find any open source marketing data so I could get caught up on AI/ML and agentic uses so here I am building a dataset that will hopefully fill that gap. When/where possible I'll use AI to generate _hopefully_ realistic text rather than use lorem ipsum. So for example, product reviews, product and campaign names etc...

If the project helped you, star the project I guess?

# Goals

Allow the user to generate as much marketing data as they need.

I'll consider this feature complete when I hit the following:

**Allow the user to generate**

- Number of years to create data for
  - Start date of sales data
- Marketing campaigns
  - Number of child campaigns that would "rollup" to a parent
  - User sentiment for each campaign (TBD)
    - Distribution of sentiment (Good, neutral, bad)
    - Generate fake reviews based on sentiment
  - Select which product categories belong to a campaign
  - Range for the duration of campaigns
  - Range for marketing budget
- Impressions
  - Used to calculate Click-through-rates
  - Whether or not an impression was shown to a user and when
  - Links to a product engagement and campaign
- Products
  - Create fake products by product category and type
    - Electronics - Mobile, Electronics - TV, Clothing - shirt...
    - When creating products most of them need to be in the "cheaper" range with a few in the more "expensive" range
  - Min/Max for product price
  - Number of product types to create per category
- Product engagement
  - Total number of individual product engagements to generate
  - Select a distribution for number of sales attributed to a campaign
  - Link sales of a product back to the campaign
  - Product reviews
    - Select distribution for reviews
    - Generate fake review text
    - Rating between 1-5

**Product categories and children**

A static set of product categories and types. This will help "seed" which products are created.

# Schema

**Marketing campaign**

- ID
- Parent campaign ID (if applicable)
- Campaign name
  - Generated by the LLM
- Budget (if a parent then the total)
- Revenue generated (if a parent then the total)
- ROI / Net revenue (if a parent then the total)
- Number of clicks (if a parent then the total)
- Conversions (units sold, if a parent then the total)
- Conversion rate (if a parent then the total)
- Product category
- Product type

**Impression**

Notes:

- Default distribution: Log-Normal

- ID
- Campaign ID
- Clicked: Beta distribution
- Date user saw

**Product Category**

- ID
- Category
- Total number of products
- Total revenue
- Units sold

**Product Type**

- ID
- Category ID
- Product count
- Revenue
- Units sold

**Product**

- ID
- Name
- Description
- Price
- Overall rating

**Product Engagement**
Notes:

- Converted if there's a sale price it was converted
- Review only happens if a sale occurred

- ID
- Engagement Date
- Product review: Bernoulli (whether or not a user left a review)
  - Rating: Power Law or Bimodal, null means no rating or review
  - Fake review: Bernoulli weighted if 1, 5 stars then more likely to leave a review
- Sale Price (Bernoulli distribution, null = user didn't purchase)
- Added to cart: Bernoulli
- Impression ID, if null user found product organically
